<Heading>TypeScript strictness</Heading>
<Spacer size="small" />
TypeScript `strict` mode enables a set of stricter type checking options.
However there are additional flags that can help improve code quality, here are
few of them.

<StylishList type="bullet" gap="loose">
  <StylishListItem>
    <code>noImplicitReturns</code>: Ensures all code paths in a function
    explicitly return a value
  </StylishListItem>
  <StylishListItem>
    <code>noUnusedLocals</code>: Flags unused local variables
  </StylishListItem>
  <StylishListItem>
    <code>noUnusedPararmetes</code>: Flags unused function parameters
  </StylishListItem>
  <StylishListItem>
    <code>noUncheckedIndexedAccess</code>: Makes indexed access return `T |
    undefined` instead of `T`
  </StylishListItem>
  <StylishListItem>
    <code>exactOptionalPropertyTypes</code>: Distinguishes between `undefined`
    and missing porperties
  </StylishListItem>
</StylishList>
